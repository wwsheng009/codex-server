# 布局系统 (Layout System)

## 1. 核心布局原语 (Layout Primitives)

在 Codex 中，所有布局必须遵循原子化的空间逻辑，避免使用硬编码的 `margin` 或 `padding`。

### 1.1 Stack (纵向律动)
用于建立垂直节奏。组件按顺序向下排列，间距由父容器统一控制。
- **使用场景**: 表单项、侧边栏菜单、文章段落、详情面板。
- **规则**: 严禁在子元素上设置 `margin-top`，必须通过父级的 `gap` 属性定义。

### 1.2 Cluster (横向分组)
用于建立水平关联。组件按行排列，支持自动换行。
- **使用场景**: 标签组 (Tags)、工具栏按钮、面包屑导航。
- **规则**: 默认水平居中对齐 (`align-items: center`)，间距应小于 Stack 的默认间距以体现视觉关联。

---

## 2. 8pt 网格与间距 Token (Spacing Grid)

系统基于 **8px (0.5rem)** 步进，确保视觉上的严谨性与比例协调。

| Token | 值 (px) | 典型用途 |
| :--- | :--- | :--- |
| `--space-xs` | 4px | 极小间距（如：图标与文字之间） |
| `--space-sm` | 8px | 标准间距（如：按钮内部 Padding） |
| `--space-md` | 16px | 组合间距（如：表单项之间的 Gap） |
| `--space-lg` | 24px | 容器内补白（如：卡片的 Padding） |
| `--space-xl` | 32px | 模块间距（如：页面大区块之间的间距） |

---

## 3. 对齐与视觉平衡 (Alignment & Balance)

### 3.1 视觉中心对齐 (Optical Alignment)
- **图标中心化**: 当图标与文字并排时，图标应基于其几何中心而非包围盒对齐。
- **悬挂标点**: 在大段文本布局中，列表符号或引号应“悬挂”在文本流之外，保持左侧视觉直线。

### 3.2 基线对齐 (Baseline Alignment)
- 不同字号的文本并排时（如：标题与副标题），必须基于 **字母基线 (Baseline)** 对齐，而非容器底部对齐。

---

## 4. 容器与层级 (Containment & Layering)

通过背景色亮度（Luminance）和阴影来划分逻辑区域，而非单纯依靠边框。

| 层级 (Layer) | 背景 Token | 视觉特征 | 用途 |
| :--- | :--- | :--- | :--- |
| **Level 0** | `--bg-layout` | 最暗/底层 | 整个页面的底色 |
| **Level 1** | `--bg-container` | 标准亮度 | 侧边栏、工具栏、基础卡片 |
| **Level 2** | `--bg-elevated` | 高亮 | 模态框、悬浮菜单、选中态 |

### 分隔规则 (Grouping Rules)
1. **优先使用留白**: 两个区块功能差异大时，增加 `gap`。
2. **次选使用背景色**: 区分导航区与工作区。
3. **末选使用边框**: 仅在背景色相同时用于划分极小颗粒度的元素（如：单元格）。

---

## 5. 响应式布局策略 (Responsive Strategy)

### 5.1 组件级流式行为 (Component Fluidity)
- **Container Queries**: 优先使用容器查询而非媒体查询。组件应根据其父容器宽度调整内部布局（如：从 Cluster 切换为 Stack），而非依赖屏幕宽度。
- **Min-Max 约束**: 所有输入控件必须定义 `min-width` 以防止在窄屏下坍塌。

### 5.2 屏幕级断点 (Fixed Breakpoints)
| 断点 | 范围 | 行为 |
| :--- | :--- | :--- |
| **Mobile** | < 768px | 侧边栏隐藏，变为抽屉；Stage 占据 100% 宽度。 |
| **Tablet** | 768px - 1280px | 侧边栏折叠为图标模式。 |
| **Desktop** | > 1280px | 完整布局，支持多列分屏。 |

---
*所有布局修改必须通过 `flex` 或 `grid` 实现，严禁使用 `float` 或绝对定位进行 UI 排版。*
